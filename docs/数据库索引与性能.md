# 数据库索引与查询性能

## MySQL 索引脚本

为加速常用查询，请在 MySQL 中执行索引脚本（首次部署或数据量增大后执行一次即可）：

```bash
# 在项目根目录执行（需已安装 MySQL 客户端并配置连接）
mysql -u 用户名 -p 数据库名 < backend/scripts/mysql-indexes.sql
```

脚本位置：`backend/scripts/mysql-indexes.sql`。若部分索引已存在会报错，可忽略或先查询 `information_schema.statistics` 再决定是否创建。

## 索引说明

| 表名 | 索引 | 用途 |
|------|------|------|
| users | idx_users_username | 登录按用户名查询 |
| users | idx_users_role, idx_users_status | 按角色、状态筛选 |
| classes | idx_classes_teacher_id | 按教师查班级 |
| assignments | idx_assignments_class_id, idx_assignments_status 等 | 作业列表分页与筛选 |
| questions | idx_questions_assignment_id | 按作业查题目 |
| submissions | idx_submissions_assignment_id, idx_submissions_student_id 等 | 提交列表与去重 |

## 慢查询配置

后端通过环境变量 `DB_SLOW_QUERY_MS` 控制慢查询日志阈值（默认 500ms）。超过该时间的 SQL 会输出到控制台，便于优化。

```env
DB_SLOW_QUERY_MS=300
```

## 连接池

`backend/src/config/database.ts` 中已配置连接池（默认 `DB_CONNECTION_LIMIT=20`），可根据并发量调整。
