# 测试数据生成脚本修改文档

## 1. 修改背景
在前期测试阶段，系统提供了一个 `create-test-users.mjs` 脚本用于在 MongoDB 数据库中快速生成不同角色的测试账号（学生、教师、家长、VIP学生）。该脚本原始逻辑会在运行前使用 `User.deleteMany({})` 清空整个 `users` 集合。这导致用户在正式或后续测试环境中使用时，已有的其他账号数据被误删，从而导致无法登录的严重问题。

根据用户的最新要求，需要将这些测试用户永久保留在数据库中，并 **严禁再次清空数据库**。

## 2. 修改文件详情
- **文件路径**：`d:\edu-ai-platform-web\Wisdom-education-platform\learning-ai-platform\server\create-test-users.mjs`

### 2.1 移除清空操作
去除了脚本开头阶段对 `User` 集合的暴力删除代码：
```javascript
// 修改前：
// 清除现有用户
await User.deleteMany({});
console.log('已清除现有用户\n');

// 修改后：
// 移除清除现有用户的操作，改为保留现有数据
// await User.deleteMany({});
console.log('保留现有用户数据，仅追加缺失的测试账号\n');
```

### 2.2 增加条件判断追加机制（Upsert 逻辑）
为防止重复运行脚本导致同一邮箱重复注册报错（或生成重复数据），将批量创建逻辑修改为：在插入每一个用户前，先通过邮箱 `User.findOne({ email: u.email })` 检查用户是否已经存在。如果存在则跳过，如果不存在则安全追加。

```javascript
// 修改前：
for (const u of users) {
  const user = new User(u);
  await user.save(); 
  console.log(`创建用户: ${u.email} (${u.role})`);
}

// 修改后：
for (const u of users) {
  const existingUser = await User.findOne({ email: u.email });
  if (existingUser) {
    console.log(`测试用户已存在，跳过创建: ${u.email}`);
  } else {
    const user = new User(u);
    await user.save(); // 触发 pre('save') hook 自动 hash 密码
    console.log(`成功追加创建测试用户: ${u.email} (${u.role})`);
  }
}
```

数据库未被清空，原数据得以安全保留，且测试账号已持久化存储于 MongoDB 中，保证了用户后续的正常登录。
