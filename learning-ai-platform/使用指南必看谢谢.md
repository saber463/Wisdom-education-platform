# 学习路径生成系统 - 使用指南

## 🔧 前提条件
- 安装 Node.js（建议版本 v16.x 或更高）
- 下载地址：https://nodejs.org/ （选择LTS长期支持版）

## 🚀 快速启动（自动安装依赖）
### Windows用户：
1. 解压项目压缩包
2. 找到项目根目录中的 `Windows自动安装依赖.bat` 文件
3. 双击 `Windows自动安装依赖.bat` 运行（会自动安装依赖，无需手动输命令）
4. 安装完成后，按提示分别启动前端和后端服务

### macOS/Linux用户：
1. 解压项目压缩包
2. 打开终端，进入项目根目录（例如：`cd 下载/你的项目文件夹`）
3. 执行命令赋予脚本权限：`chmod +x mac自动安装依赖.sh`
4. 运行脚本自动安装依赖：`./mac自动安装依赖.sh`
5. 安装完成后，按提示分别启动前端和后端服务

## 📦 核心命令
| 命令          | 作用                  |
|---------------|-----------------------|
| `npm run dev` | 启动开发环境（前端/后端）|
| `npm run build` | 打包生产环境（前端部署用）|
| `npm install` | 手动安装依赖（脚本失败时用）|

## 🖥️ 启动项目（需分别启动前端和后端）
### 启动前端（client）：
1. 打开终端，进入项目根目录下的 `client` 文件夹：`cd client`
2. 执行命令：`npm run dev`
3. 启动成功后，访问：http://localhost:3000

### 启动后端（server）：
1. 打开另一个终端，进入项目根目录下的 `server` 文件夹：`cd server`
2. 执行命令：`npm run dev`
3. 启动成功后，后端服务将在 http://localhost:3001 运行

## ❌ 常见问题排查
1. **脚本双击没反应/报错**：
   - 检查Node.js是否安装成功（终端输入 `node -v` 能显示版本即可）
   - Windows用户若提示“权限不足”，右键脚本文件选择“以管理员身份运行”

2. **依赖安装缓慢/失败**：
   - 确保网络通畅，国内用户建议使用淘宝镜像（脚本已自动配置）
   - 手动切换镜像：`npm config set registry https://registry.npmmirror.com/`
   - 重试安装：`npm install`

3. **启动项目报错“模块缺失”**：
   - 删除 `node_modules` 文件夹和 `package-lock.json`
   - 重新执行安装脚本，或手动运行 `npm install`

4. **后端启动失败**：
   - 确保MongoDB已安装并运行
   - 检查 `.env` 文件中的数据库配置是否正确
