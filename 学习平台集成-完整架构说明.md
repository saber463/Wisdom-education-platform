## 🎯 核心功能详解

### 1. 课程体系

**三层结构**:
```
编程语言（Course）
  └─ 分支（Branch）- 如Python的Web开发、数据分析、机器学习
      └─ 课节（Lesson）- 每节课的视频、内容、练习
```

**课程购买流程**:
1. 学生浏览课程 → 查看详情 → 购买课程
2. 系统随机分配到该分支的6个班级之一（负载均衡）
3. 创建 `course_purchases` 记录
4. 创建 `class_students` 关联
5. 通知学生和教师

**模板教师**:
- 系统预设固定教师账号
- 每个课程分支创建6个班级
- 每个班级分配一个模板教师
- 教师可以查看班级学生、发布公告、创建作业

### 2. 视频学习追踪

**实时追踪**:
- 每5秒记录一次播放位置到MongoDB
- 记录暂停位置和暂停时长
- 检测快进、跳过、挂机行为

**教师视图**:
```typescript
// 教师可以看到每个学生的：
{
  student_name: "张三",
  video_progress: 85%,        // 观看进度
  watch_count: 3,             // 观看次数
  total_watch_time: "45分钟", // 总观看时长
  avg_speed: 1.5,             // 平均播放速度
  heat_map: [                 // 热力图
    { start: 120, end: 180, count: 3 }  // 2-3分钟被重复观看3次
  ],
  abnormal: false             // 是否有异常行为
}
```

### 3. 个性化推荐系统

**推荐算法**:

**协同过滤** (Collaborative Filtering):
```typescript
// 基于用户相似度推荐
// "学过课程A的用户也学过课程B"
async function collaborativeFiltering(userId: number) {
  // 1. 找到相似用户（学习历史相似）
  const similarUsers = await findSimilarUsers(userId);
  
  // 2. 获取相似用户学过但当前用户没学过的课程
  const recommendations = await getCoursesFromSimilarUsers(similarUsers, userId);
  
  // 3. 按相似度和课程评分排序
  return recommendations.sort((a, b) => b.score - a.score);
}
```

**内容推荐** (Content-Based):
```typescript
// 基于课程内容相似度推荐
async function contentBasedRecommendation(userId: number) {
  // 1. 获取用户已学课程的标签和知识点
  const userProfile = await getUserLearningProfile(userId);
  
  // 2. 计算其他课程与用户画像的相似度
  const courses = await getAllCourses();
  const scored = courses.map(course => ({
    ...course,
    similarity: calculateSimilarity(userProfile, course.tags)
  }));
  
  // 3. 返回相似度最高的课程
  return scored.sort((a, b) => b.similarity - a.similarity);
}
```

**混合推荐** (Hybrid):
```typescript
// 综合多种算法
async function hybridRecommendation(userId: number) {
  const cf = await collaborativeFiltering(userId);
  const cb = await contentBasedRecommendation(userId);
  const hot = await getHotCourses();
  
  // 加权融合：协同过滤40% + 内容推荐40% + 热门课程20%
  return mergeRecommendations(cf, cb, hot, [0.4, 0.4, 0.2]);
}
```

**推荐触发时机**:
- 用户首次登录：基于注册时选择的兴趣
- 完成课程后：推荐进阶课程
- 浏览课程时：实时推荐相关课程
- 每天凌晨：重新计算推荐结果并缓存到MongoDB

### 数据关联

**学习路径 ↔ 作业**:
- 学习路径的某个步骤可以关联作业
- 学生学习到该步骤时，可以直接跳转到作业提交
- 完成作业后，自动更新学习路径进度

**学习资源 ↔ 知识点**:
- 学习资源标记知识点
- 学生在作业中遇到薄弱知识点时，推荐相关资源
- 教师创建学习路径时，根据知识点筛选资源

**课程 ↔ 班级 ↔ 作业**:
- 购买课程后自动加入班级
- 教师可以为班级创建作业
- 作业可以关联到课程的某个课节
