## 快速开始

### 方式1：一键启动（推荐）

```batch
# 1. 克隆项目
git clone https://github.com/your-repo/smart-education-platform.git
cd smart-education-platform

# 2. 双击运行一键启动脚本
scripts/start-all-services.bat

# 3. 等待10秒，浏览器自动打开
# 默认地址：http://localhost:5173
```

### 方式2：手动启动

```batch
# 1. 启动MySQL
net start MySQL80

# 2. 启动Rust服务
cd rust-service
cargo run --release

# 3. 启动Python服务
cd python-ai
python grpc_server.py

# 4. 启动Node后端
cd backend
npm start

# 5. 启动前端
cd frontend
npm run dev
```

## 详细安装步骤

### 步骤1：安装Node.js

1. 访问 https://nodejs.org/
2. 下载18.x LTS版本（Windows 64位安装包）
3. 运行安装程序，勾选"Add to PATH"
4. 验证安装：
   ```batch
   node --version  # 应显示 v18.x.x
   npm --version   # 应显示 9.x.x
   ```

### 步骤2：安装Python

1. 访问 https://www.python.org/downloads/
2. 下载Python 3.10+版本
3. 运行安装程序，**务必勾选"Add Python to PATH"**
4. 验证安装：
   ```batch
   python --version  # 应显示 Python 3.10.x
   pip --version     # 应显示 pip 23.x.x
   ```

### 步骤3：安装Rust

1. 访问 https://www.rust-lang.org/tools/install
2. 下载rustup-init.exe
3. 运行安装程序，选择默认安装
4. 验证安装：
   ```batch
   rustc --version  # 应显示 rustc 1.7x.x
   cargo --version  # 应显示 cargo 1.7x.x
   ```

### 步骤4：安装MySQL

#### 方式A：使用Navicat（推荐）

1. 安装Navicat Premium
2. 系统会自动检测Navicat并创建数据库
3. 无需手动配置

#### 方式B：手动安装MySQL

1. 访问 https://dev.mysql.com/downloads/mysql/
2. 下载MySQL 8.0 Windows安装包
3. 运行安装程序：
   - 选择"Developer Default"
   - 设置root密码（建议留空或设置为123456）
   - 端口保持默认3306
4. 验证安装：
   ```batch
   mysql --version  # 应显示 mysql Ver 8.0.x
   ```

#### 方式C：自动安装（系统提供）

```batch
# 运行自动安装脚本
cd backend/scripts
node install-mysql.cjs
```

### 步骤5：安装项目依赖

```batch
# 1. 安装Node.js依赖
cd backend
npm install

cd ../frontend
npm install

# 2. 安装Python依赖
cd ../python-ai
pip install -r requirements.txt

# 3. 编译Rust项目
cd ../rust-service
cargo build --release

cd ../rust-wasm
wasm-pack build --target web --release
```

### 步骤6：初始化数据库

```batch
# 方式A：自动初始化（推荐）
cd backend/scripts
node setup-database.cjs

# 方式B：手动初始化
mysql -u root -p < docs/sql/schema.sql
mysql -u root -p < docs/sql/test-data.sql
```

## 一键启动说明

### 启动脚本清单

| 脚本名称 | 用途 | 位置 |
|---------|------|------|
| **start-all-services.bat** | 正常启动所有服务 | scripts/ |
| **start-lightweight-mode.bat** | 轻量运行模式（防蓝屏） | scripts/ |
| **stop-all-services.bat** | 停止所有服务 | scripts/ |
| **emergency-repair.bat** | 应急修复 | scripts/ |
| **reset-demo-data.bat** | 重置演示数据 | scripts/ |
| **backup-database.bat** | 备份数据库 | scripts/ |

### 正常启动流程

```batch
# 双击运行
scripts/start-all-services.bat
```

**启动顺序**：
1. MySQL数据库（端口3306）
2. Rust高性能服务（端口8080）
3. Python AI服务（端口5000）
4. Node.js后端（端口3000）
5. Vue前端（端口5173）

**预计耗时**：8-12秒

**启动成功标志**：
- 浏览器自动打开 http://localhost:5173
- 显示登录页面
- 控制台显示"所有服务启动成功"

### 轻量运行模式

```batch
# 双击运行
scripts/start-lightweight-mode.bat
```

**特点**：
- CPU使用率限制≤50%
- 内存使用率限制≤50%
- 服务以低优先级运行
- 适合配置较低的电脑

### 停止服务

```batch
# 双击运行
scripts/stop-all-services.bat
```

**停止顺序**：
1. 前端服务
2. Node.js后端
3. Python AI服务
4. Rust服务
5. MySQL数据库

**预计耗时**：3-5秒

## 故障排查指南

### 问题1：端口被占用

**现象**：
```
Error: listen EADDRINUSE: address already in use :::3000
```

**原因**：端口3000/3306/5000/8080被其他程序占用

**解决方案**：

方式A：自动切换端口
```batch
# 系统会自动切换到备用端口
# 3000 → 3001 → 3002
# 3306 → 3307 → 3308
# 5000 → 5001 → 5002
# 8080 → 8081 → 8082
```

方式B：手动释放端口
```batch
# 查找占用端口的进程
netstat -ano | findstr :3000

# 结束进程（PID为查找到的进程ID）
taskkill /PID <PID> /F
```

方式C：运行应急修复脚本
```batch
scripts/emergency-repair.bat
```

### 问题2：MySQL连接失败

**现象**：
```
Error: connect ECONNREFUSED 127.0.0.1:3306
```

**原因**：MySQL服务未启动或连接配置错误

**解决方案**：

方式A：启动MySQL服务
```batch
net start MySQL80
```

方式B：检查MySQL配置
```javascript
// backend/src/config/database.ts
const dbConfig = {
  host: 'localhost',
  port: 3306,
  user: 'root',
  password: '',  // 检查密码是否正确
  database: 'edu_education_platform'
};
```

方式C：重新安装MySQL
```batch
cd backend/scripts
node install-mysql.cjs
```

### 问题3：Python依赖安装失败

**现象**：
```
ERROR: Could not find a version that satisfies the requirement torch
```

**原因**：网络问题或镜像源不可用

**解决方案**：

方式A：切换清华镜像源
```batch
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
pip install -r requirements.txt
```

方式B：使用国内镜像安装
```batch
pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple/
```

方式C：手动下载whl文件
```batch
# 访问 https://download.pytorch.org/whl/torch_stable.html
# 下载对应版本的whl文件
pip install torch-2.1.2+cpu-cp310-cp310-win_amd64.whl
```

### 问题4：Rust编译失败

**现象**：
```
error: linking with `link.exe` failed: exit code: 1120
```

**原因**：内存不足或编译器错误

**解决方案**：

方式A：单核编译（防止内存溢出）
```batch
set CARGO_BUILD_JOBS=1
cargo build --release
```

方式B：清理缓存重新编译
```batch
cargo clean
cargo build --release
```

方式C：降低优化级别
```toml
# Cargo.toml
[profile.release]
opt-level = 2  # 从3降到2
```

### 问题5：WASM模块加载失败

**现象**：
```
Failed to load WASM module
```

**原因**：WASM文件未编译或路径错误

**解决方案**：

方式A：重新编译WASM
```batch
cd rust-wasm
wasm-pack build --target web --release
copy pkg\* ..\frontend\src\wasm\
```

方式B：检查文件路径
```typescript
// frontend/src/utils/wasm-loader.ts
import init from '@/wasm/edu_wasm';  // 确认路径正确
```

### 问题6：BERT模型下载失败

**现象**：
```
OSError: Can't load tokenizer for 'bert-base-chinese'
```

**原因**：网络问题，无法从Hugging Face下载模型

**解决方案**：

方式A：使用国内镜像
```python
# python-ai/bert_grading.py
from transformers import BertModel, BertTokenizer

# 设置镜像源
os.environ['HF_ENDPOINT'] = 'https://hf-mirror.com'

model = BertModel.from_pretrained('bert-base-chinese')
```

方式B：手动下载模型
```batch
# 1. 访问 https://hf-mirror.com/bert-base-chinese
# 2. 下载所有文件到 python-ai/models/bert-base-chinese/
# 3. 修改代码使用本地路径
model = BertModel.from_pretrained('./models/bert-base-chinese')
```

### 问题7：系统蓝屏

**现象**：Windows蓝屏重启

**原因**：CPU/内存过载

**预防措施**：
```batch
# 使用轻量运行模式
scripts/start-lightweight-mode.bat
```

**恢复措施**：
```batch
# 系统重启后运行蓝屏恢复脚本
scripts/blue-screen-recovery.bat
```

### 问题8：前端页面空白

**现象**：浏览器显示空白页面

**原因**：前端服务未启动或构建失败

**解决方案**：

方式A：检查前端服务
```batch
cd frontend
npm run dev
```

方式B：清理缓存重新构建
```batch
cd frontend
rmdir /s /q node_modules
rmdir /s /q dist
npm install
npm run dev
```

方式C：检查浏览器控制台
```
按F12打开开发者工具，查看Console和Network标签页
```

## 常见问题FAQ

### Q1：系统支持哪些操作系统？

**A**：目前仅支持Windows 10/11 (64位)。不支持macOS和Linux。

### Q2：可以使用Docker部署吗？

**A**：不可以。本项目专为Windows本地环境设计，禁用Docker/容器化。

### Q3：数据库密码是什么？

**A**：默认root密码为空。如果设置了密码，请修改`backend/src/config/database.ts`。

### Q4：如何修改服务端口？

**A**：修改对应服务的配置文件：
- Node.js: `backend/src/index.ts`
- Python: `python-ai/config.py`
- Rust: `rust-service/src/main.rs`
- 前端: `frontend/vite.config.ts`

### Q5：如何重置演示数据？

**A**：运行重置脚本：
```batch
scripts/reset-demo-data.bat
```

### Q6：系统启动需要多长时间？

**A**：正常情况下8-12秒。首次启动可能需要15-20秒（需要下载模型）。

### Q7：如何备份数据库？

**A**：运行备份脚本：
```batch
scripts/backup-database.bat
```
备份文件保存在`docs/sql/backup/`目录。

### Q8：如何更新系统？

**A**：
```batch
# 1. 停止所有服务
scripts/stop-all-services.bat

# 2. 拉取最新代码
git pull

# 3. 更新依赖
cd backend && npm install
cd ../frontend && npm install
cd ../python-ai && pip install -r requirements.txt

# 4. 重新启动
scripts/start-all-services.bat
```

### Q9：系统占用多少资源？

**A**：
- CPU: 30-50%（正常模式），20-30%（轻量模式）
- 内存: 4-6GB（正常模式），2-3GB（轻量模式）
- 硬盘: 约15GB（包含依赖和模型）

### Q10：如何卸载系统？

**A**：
```batch
# 1. 停止所有服务
scripts/stop-all-services.bat

# 2. 删除项目文件夹
rmdir /s /q smart-education-platform

# 3. 卸载MySQL（可选）
# 通过控制面板卸载MySQL Server

# 4. 卸载Node.js/Python/Rust（可选）
# 通过控制面板卸载
```

## 性能优化建议

CREATE INDEX idx_assignment_class ON assignments(class_id);
CREATE INDEX idx_submission_student ON submissions(student_id);

DELETE FROM system_logs WHERE created_at < DATE_SUB(NOW(), INTERVAL 30 DAY);

// 启用Gzip压缩

## 安全建议

### 1. 修改默认密码

```javascript
// backend/src/config/database.ts
password: 'your-strong-password'  // 不要使用空密码
```

### 2. 启用HTTPS

```javascript
// backend/src/index.ts
const https = require('https');
const fs = require('fs');

const options = {
  key: fs.readFileSync('ssl/private.key'),
  cert: fs.readFileSync('ssl/certificate.crt')
};

https.createServer(options, app).listen(3000);
```

### 3. 配置防火墙

```batch
# 只允许本地访问
netsh advfirewall firewall add rule name="MySQL" dir=in action=allow protocol=TCP localport=3306 remoteip=127.0.0.1
```
