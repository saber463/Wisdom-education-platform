# 项目修改说明文档

## 1. 修改背景
为了提升智慧教育平台的用户体验和内容专业度，对前端兴趣选择弹窗进行了文案优化，并同步更新了后端推文数据的生成逻辑，确保推文配图更加专业化。

## 2. 修改内容详解

### 2.1 前端修改 (兴趣选择弹窗)
**修改目标**：在用户初次登录时的兴趣选择弹窗中增加更前沿的技术领域文案，特别是机器学习和大语言模型相关内容。

**涉及文件**：
- `client/src/components/common/InterestPopup.vue`
- `client/src/components/common/InterestModal.vue`

**详细变更**：
- 在 `InterestPopup.vue` 的 `interestTags` 列表中新增了以下标签：
  - `深度学习`
  - `自然语言处理`
  - `计算机视觉`
  - `大语言模型`
- 在 `InterestModal.vue` 的可视化兴趣卡片中新增了两个分类：
  - **机器学习**：描述为“监督学习, 无监督学习, 强化学习”
  - **大语言模型**：描述为“GPT-4, Claude, 文心一言, 智能对话”

### 2.2 后端修改 (推文数据生成脚本)
**修改目标**：修复系统自动生成的推文配图，将其从随机图片改为对应技术的官方 Logo，提升社区内容的专业观感。

**涉及文件**：
- `server/scripts/generate-system-tweets.js`

**详细变更**：
- **ESM 模块化转换**：将脚本从 CommonJS 格式转换为 ESM 格式（使用 `import` 代替 `require`），以适配项目的 `type: module` 配置。
- **Logo 映射表**：新增了 `techLogos` 常量，建立了技术标签（如：前端开发、Python、机器学习等）与对应官方 SVG Logo 地址（来源于 `devicon` CDN）的映射关系。
- **生成逻辑更新**：
  - 修改 `generateSystemTweets` 函数，在生成推文时根据话题标签自动匹配对应的官方 Logo。
  - 增加了自动清理逻辑：脚本运行前会先清除旧的系统推文，确保所有推文都更新为带有固定 Logo 的新版本。
  - 每种技术标签现在固定生成 50 条带有对应图标的专业推文。

## 3. 个性化推荐算法
为了实现“千人千面”的学习体验，本次更新在推文社区和知识库中均引入了基于用户兴趣标签的推荐算法。

### 3.1 核心算法逻辑
- **标签匹配**：系统提取用户在个人中心或登录弹窗中选择的 `learningInterests`（学习兴趣）标签。
- **内容过滤**：后端接口通过 `$in` 操作符，在数据库中检索包含这些兴趣标签的推文或文章。
- **前端重排**：若后端接口未完全过滤，前端会根据匹配权重对内容进行二次排序。
  - **标题匹配**：权重 3.0
  - **关键词匹配**：权重 2.5
  - **分类匹配**：权重 2.0
  - **摘要/内容匹配**：权重 1.0~1.5

### 3.2 模块落地
- **学习社区**：
  - 更新 `getTweets` 接口支持 `recommended=true` 参数。
  - 推文列表优先展示与用户兴趣标签（如“机器学习”、“前端开发”）一致的动态。
- **知识库**：
  - 更新 `getArticles` 接口，强制要求 `auth` 认证以获取用户兴趣。
  - 文章列表通过 `calculateMatchScore` 辅助函数实现基于兴趣的高分内容置顶。

## 4. AI 学习路径文案优化
为了提升演示效果，针对高频搜索场景“计算机一级”进行了专门的路径文案优化。

### 4.1 核心改进
- **内置精品模板**：在 `xunfeiRoadmapService.js` 中新增了 `ncre_level1` 专业模板。
- **30天科学规划**：将备考分为 6 个阶段（理论基础、Windows、Word、Excel、PPT、模拟冲刺）。
- **关键词精准触发**：后端增加了对“计算机一级”、“Office”、“NCRE”、“WPS”等关键词的智能识别，优先匹配高质量内置路径。

## 5. 执行结果
- 脚本已成功执行，数据库中的系统推文已全部更新。
- 前端弹窗已同步更新，用户现在可以选择更细分的 AI 技术领域。

## 4. 后续维护
- 如需添加新的技术领域，请同步更新 `server/scripts/generate-system-tweets.js` 中的 `interestTags` 和 `techLogos` 映射。
- 运行脚本前请确保 MongoDB 服务已启动。
