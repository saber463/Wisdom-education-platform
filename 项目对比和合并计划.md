# 项目对比和合并计划

## 项目概览

### 当前项目 (edu-ai-platform-web)
- **名称**: 智慧教育学习平台
- **前端**: Vue 3 + TypeScript + Element Plus (端口: 5173)
- **后端**: Node.js + Express + TypeScript (端口: 3000)
- **数据库**: MySQL
- **AI服务**: Python gRPC (端口: 50051)
- **特点**: 
  - 三角色系统（教师、学生、家长）
  - 作业管理、批改、学情分析
  - 分层教学、AI答疑
  - 口语评测、学习小组

### Learning-AI-Platform
- **名称**: AI LearnHub
- **前端**: Vue 3 + JavaScript + Element Plus (端口: 3000)
- **后端**: Node.js + Express + JavaScript (端口: 5000)
- **数据库**: MongoDB
- **AI服务**: 集成多种AI API (百度AI、讯飞AI、Qwen3)
- **特点**:
  - 个性化学习路径
  - AI聊天助手
  - 学习资源分享
  - 学习进度跟踪
  - 通知系统
  - 学习小组

## 数据库选择建议

### 方案A: 使用MySQL（推荐）
**优势**:
- 当前项目已有完整的MySQL schema
- 关系型数据更适合教育系统（作业、成绩、班级关系）
- 数据一致性更强

**迁移工作**:
- 需要将learning-ai-platform的MongoDB数据模型转换为MySQL
- 创建对应的表结构
- 迁移现有数据

### 方案B: 使用MongoDB
**优势**:
- learning-ai-platform已有完整的MongoDB模型
- 灵活的文档结构
- 适合快速迭代

**迁移工作**:
- 需要将edu-ai-platform-web的MySQL schema转换为MongoDB
- 重写所有数据库查询
- 工作量较大

### 方案C: 双数据库（不推荐）
- 同时使用MySQL和MongoDB
- 增加系统复杂度
- 维护成本高

**建议**: 选择方案A（MySQL），因为教育系统的关系型数据特征更明显。

## 数据库迁移计划

### 新增表结构

```sql
-- 学习路径表
CREATE TABLE learning_paths (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  title VARCHAR(255) NOT NULL,
  description TEXT,
  difficulty ENUM('beginner', 'intermediate', 'advanced'),
  status ENUM('active', 'completed', 'archived'),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 学习资源表
CREATE TABLE resources (
  id INT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(255) NOT NULL,
  description TEXT,
  type ENUM('article', 'video', 'document', 'link'),
  url VARCHAR(500),
  category_id INT,
  user_id INT NOT NULL,
  is_public BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 通知表
CREATE TABLE notifications (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  type ENUM('system', 'assignment', 'grade', 'message'),
  title VARCHAR(255) NOT NULL,
  content TEXT,
  is_read BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 学习进度表
CREATE TABLE learning_progress (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL,
  learning_path_id INT NOT NULL,
  progress INT DEFAULT 0,
  last_accessed_at TIMESTAMP,
  completed_at TIMESTAMP NULL,
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (learning_path_id) REFERENCES learning_paths(id)
);
```

## 风险评估

### 高风险项
1. **数据库迁移**: MongoDB到MySQL的转换可能丢失数据
   - **缓解措施**: 充分测试，保留MongoDB备份

2. **API兼容性**: 两个项目的API可能冲突
   - **缓解措施**: 统一API设计，使用版本控制

3. **性能问题**: 合并后系统可能变慢
   - **缓解措施**: 性能测试，优化查询

### 中风险项
1. **代码冲突**: 相同功能的不同实现
   - **缓解措施**: 代码审查，选择最优实现

2. **依赖冲突**: 不同版本的依赖包
   - **缓解措施**: 统一依赖版本

### 低风险项
1. **样式冲突**: CSS样式可能冲突
   - **缓解措施**: 使用Tailwind CSS统一样式
