# 登录失败问题修复记录

## 1. 问题现象
用户在前端页面无法正常登录，页面提示“用户名或密码错误”或服务无响应。

## 2. 问题排查过程
1. **查阅文档**：首先读取了项目根目录下的相关说明文档，包括 `测试账号说明.md`、`当前问题诊断和解决方案.md` 等，明确了系统内置的测试账号信息（如教师账号 `teacher001/teacher123`）。
2. **后端服务状态检查**：
   - 发现 Node.js 后端服务由于端口 3000 冲突导致启动失败。
   - 强制结束了残留的 Node 进程并重新启动服务。
   - 在启动后测试登录接口，发现服务抛出 `Cannot set headers after they are sent to the client` 的异常并崩溃。
3. **数据库状态检查**：
   - 检查 `edu_education_platform` 数据库，发现 `users` 表内没有任何数据，导致所有的登录请求都会返回“用户名或密码错误”。

## 3. 修复方案及执行细节
1. **修复后端服务异常崩溃**
   - **涉及文件**：`backend/src/middleware/performance.ts`
   - **修改内容**：在性能监控中间件 `performanceMonitor` 的 `res.on('finish', ...)` 回调中，由于响应已经结束（`finish` 事件触发时 header 已发送），不能再调用 `res.setHeader('X-Response-Time', ...)` 否则会引发 Node.js 崩溃。将其注释处理。
2. **重新导入测试数据**
   - 编写并执行了 Node.js 脚本，读取了 `docs/sql/test-data.sql` 文件并重新向 MySQL 数据库的 `edu_education_platform` 库中导入了所有测试用户数据。
3. **重启并验证服务**
   - 重启后端服务运行在 3000 端口，前端 Vite 服务运行在 5173 端口。
   - 使用 `teacher001` 和 `teacher123` 请求 `/api/auth/login` 接口验证，目前已成功返回包含 `token` 的 JSON 数据，登录恢复正常。

## 4. 后续建议
后续开发过程中如果数据库数据被清空，可以使用 `docs/sql/test-data.sql` 重新填充；对于 Express 中间件的事件监听，应避免在 `finish` 事件后进行响应头的设置。
