6. 定位根本原因：`connection.execute()` 对 `LIMIT ? OFFSET ?` 支持不完善

**关键发现**:
```
Error: Incorrect arguments to mysqld_stmt_execute
```

### 2. 修复数据库查询问题 ✅

**修改文件**: `backend/src/config/database.ts`

**修改内容**:
```typescript
// 修改前
const [results] = await connection.execute(sql, params);

// 修改后
const [results] = await connection.query(sql, params);
```

**原因**:
- `execute()` 使用MySQL预处理语句，某些版本对LIMIT/OFFSET参数绑定支持不完善
- `query()` 使用常规查询，兼容性更好，仍然安全（mysql2会自动转义参数）

## 📝 技术要点总结

### MySQL execute() vs query()

| 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| execute() | 性能好（预处理）<br>防SQL注入 | 某些MySQL版本对LIMIT/OFFSET支持不完善 | 频繁执行的简单查询 |
| query() | 兼容性好<br>参数绑定灵活 | 性能略低 | 复杂查询、带LIMIT/OFFSET的查询 |

**本项目选择**: `query()` - 兼容性优先，性能差异可忽略
